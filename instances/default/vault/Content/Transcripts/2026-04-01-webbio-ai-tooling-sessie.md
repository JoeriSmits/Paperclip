# 2026-04-01 Webbio MT: AI tooling sessie

## Context

Webbio MT-vergadering. Joeri presenteert evaluatie van 3 maanden AI tooling gebruik binnen het dev team (Cursor, GitHub Copilot). Discussie over wat werkt, wat niet werkt, en of we dit verder uitrollen naar meer teamleden.

## Transcript

**Joeri:** Ik heb de afgelopen 3 maanden bijgehouden hoe we Cursor gebruiken binnen het team. Wat ik zie is dat de junior developers er eigenlijk méér van leren dan de seniors. De seniors skippen de suggesties vaker, maar de juniors gebruiken het als een soort interactieve documentatie.

**MT-lid:** Maar is dat niet gevaarlijk? Dat ze blind code accepteren?

**Joeri:** Dat is precies wat ik in het begin ook dacht. Maar wat ik zie is dat de code review het vangnet is. De kwaliteit van de code reviews is juist verbeterd — je bespreekt nu de keuzes in plaats van de syntax. De discussies gaan over architectuur, niet over puntkomma's.

**MT-lid 2:** En de snelheid? Leveren we echt sneller?

**Joeri:** Op story-level? Nauwelijks. Maar de doorlooptijd van kleine bugs is gehalveerd. En de onboarding van nieuwe developers is een stuk sneller gegaan bij de laatste twee mensen die we hebben aangenomen.

**MT-lid:** Dus de ROI zit hem niet in story velocity maar in...

**Joeri:** In het vrijmaken van aandacht. De aandacht gaat nu naar het moeilijke, niet het eenvoudige. Dat is de winst.

## Key Takeaways

- AI coding tools helpen juniors meer dan seniors (reverse expectation)
- Code review kwaliteit is verbeterd: gesprekken gaan over architectuur ipv syntax
- ROI zit niet in story velocity maar in onboarding-snelheid en aandachtsverdeling
- "De aandacht gaat nu naar het moeilijke, niet het eenvoudige" — sterke one-liner
